# Alert Center

The Alert Center is the operational notification history for each org. It records alerts sent by the platform and supports filtering, export, and failure investigation.

It is not the same as:

- delivery logs
- dashboard metrics
- real-time worker health

Use Alert Center for notification history, and System Status / System Logs for runtime diagnosis.

---

## How Alerts Work

Background workers and scheduled checks generate alerts when thresholds or error conditions are met.

Each alert is:

1. generated by the platform
2. dispatched to the configured channel
3. recorded with metadata, status, and provider response

The Alert Center is therefore a history of dispatched alert activity, not a live event stream.

---

## Alert Categories

Typical categories include:

| Category | Trigger |
|----------|---------|
| Integration delivery failures | High failure rate in a configured window |
| Connection errors | Network or downstream connectivity issues |
| Transformation errors | Script or mapping failures |
| Authentication failures | Expired or invalid downstream credentials |
| Rate limiting | Excessive request throttling |
| DLQ threshold | Dead letter queue growth over threshold |
| Abandoned delivery | Final retry exhaustion |

---

## Notification Channels

| Channel | Description |
|---------|-------------|
| `EMAIL` | Email to one or more recipients |
| `SLACK` | Slack message delivery |
| `WEBHOOK` | HTTP POST to a configured endpoint |

Alert delivery is still org-scoped and logged with status and response detail.

---

## Failure Reports

Failure reports are scheduled alert summaries that scan a lookback window and notify only when the failure count crosses a threshold.

Typical fields:

| Field | Default | Description |
|-------|---------|-------------|
| `enabled` | `false` | Enables automatic reports |
| `intervalMinutes` | `15` | How often the scheduler runs |
| `lookbackMinutes` | `60` | Failure scan window |
| `minFailures` | `1` | Minimum failures before a report is sent |
| `maxItems` | `25` | Maximum failures included in one report |

---

## Using Alert Center with Other Ops Views

Recommended flow:

1. Alert Center
   - find what alert was sent
2. Delivery Logs
   - inspect the underlying failed executions
3. System Status
   - check worker, adapter, and process health
4. System Logs
   - inspect runtime/application log output

This is the intended operational split.

---

## Export and Search

Alert history can be filtered and exported by:

- status
- channel
- type
- recipient
- date range
- search text

Use exports for support follow-up and audit review.
